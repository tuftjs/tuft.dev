What?

Tuft is an HTTP/2 web framework for Node.js that has been designed to provide excellent routing performance, even when serving thousands of routes, while maintaining minimal overhead. While it keeps its feature set to a minimum, it provides mechanisms to extend its functionality so that it can be adapted to almost any need.


Why?

Tuft was built with the primary objective of becoming a viable HTTP/2 framework for Node.js. Specifically, it has been built to utilize Node's core HTTP/2 API. Not all of the most commonly used frameworks support HTTP/2, and the ones that do tend to treat it like a second-class citizen by only supporting it via Node's compatibility API.

HTTP/2 provides a significant performance increase over HTTP/1, and we are now at a stage where all major web browsers support it. Some of its benefits include:

  • Minimal overhead via efficient HTTP header compression with HPACK.
  • Reduced latency via full request/response multiplexing.
  • Increased security by favoring binary transmission over text.
  • Server push (this feature has not yet been implemented in Tuft, it's on the to-do list!).

All of the above is possible while maintaining the core concepts of HTTP, including URIs, status codes, and request methods.


When?

Development of Tuft in its first incarnation started in early 2019 as a part-time project to familiarize myself with the inner workings of Node.js. It has evolved a lot since then, but it is still untested software.
