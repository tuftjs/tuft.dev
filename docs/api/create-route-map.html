createRouteMap([options])

The Tuft library does not have a default export. Instead, it exports a named function called createRouteMap(), which is the intended method for creating a Tuft "application". It returns an instance of TuftRouteMap, to which all route data and associated response handlers are added. An options object can be passed as the only argument to set defaults for all routes that are subsequently added.

const { createRouteMap } = require('tuft')

const options = {
  trailingSlash: true,
  basePath: '/foo'
}

const app = createRouteMap(options)

The options argument may contain any of the following properties:

trailingSlashboolean

Match routes that have a trailing slash. If set to true, the route '/foo' would be matched by both '/foo' and '/foo/'.

Defaults to null.

basePathstring

Prepended to the path when adding a route. If it were set to '/foo', then adding a route with the path '/bar' would result in a route of '/foo/bar'.

If this option is not provided, then nothing will be prepended to the route path.

methodstring | string[]

The default HTTP request method(s) to be included when matching a route.

By default, all of the following supported methods will be matched:

  • GET
  • HEAD
  • POST
  • PUT
  • PATCH
  • DELETE
  • OPTIONS
  • TRACE

pathstring

Used as the default value for path if one is not provided when adding a route.

Defaults to '/'.

preHandlersTuftPreHandler[]

An array of Tuft pre-handlers that are executed serially before the response handler for each route that gets added. For more information, see the section on pre-handlers.

Defaults to [].

respondersTuftResponder[]

An array of Tuft responders that are executed serially after the response handler returns. A Tuft responder is a function which receives a response object, and then determines how to respond to the client based on the properties included in that object. For more information, see the section on Tuft responders.

Defaults to [].
